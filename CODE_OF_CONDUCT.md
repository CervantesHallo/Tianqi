# Code of Conduct

Tianqi (天启) adopts the **[Contributor Covenant Code of Conduct, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** as the standard of behavior for all participants in this project's spaces (issues, pull requests, discussions, security advisories, and any other channel associated with the project).

The full text of the Contributor Covenant 2.1 is the authoritative source. This file intentionally does not reproduce that text — please refer to the canonical version at the link above to ensure you are reading the current wording maintained by the Contributor Covenant authors.

## Scope

This Code of Conduct applies within all project spaces and also when an individual is officially representing the project in public.

## Reporting and Enforcement

Concerns, abuse reports, or questions about this Code of Conduct should be raised through one of the following channels, in order of preference:

1. **Private Security Advisory** (preferred for anything sensitive): use [GitHub Private Vulnerability Reporting](https://github.com/CervantesHallo/Tianqi/security/advisories/new). The same channel is used for security reports (see [SECURITY.md](./SECURITY.md)) — clearly mark the report as a Code of Conduct concern.
2. **GitHub Discussion or Issue** (for non-sensitive, general questions about the policy itself): open a regular Discussion or Issue in this repository.

The current project maintainer is the GitHub account that owns this repository (`@CervantesHallo`). All reports will be reviewed and handled with discretion. As Tianqi is currently maintained by a single maintainer, response times follow the same best-effort commitments documented in [SECURITY.md](./SECURITY.md).
